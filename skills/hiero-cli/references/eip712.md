# eip712 plugin

Sign and verify EIP-712 structured typed data using ECDSA or Ed25519 keys managed by the CLI KMS. Stateless — no local state is written. Useful for permit flows, off-chain authorizations, meta-transactions, and dApp integrations on Hedera.

Domain, types, and message each accept either an **inline JSON string** or a **path to a JSON file**.

For sign/verify commands you can provide either:

- **`--hash`** — a pre-computed EIP-712 digest (0x-prefixed keccak256 hex), OR
- **`--domain` + `--types` + `--message`** — the full typed data (all three required together)

These two input modes are mutually exclusive.

---

### `hcli eip712 hash`

Compute the EIP-712 digest (keccak256 hash) for a typed data payload without signing.

| Option      | Short | Type   | Required | Description                                                    |
| ----------- | ----- | ------ | -------- | -------------------------------------------------------------- |
| `--domain`  | `-d`  | string | **yes**  | EIP-712 domain as inline JSON or path to a JSON file           |
| `--types`   | `-t`  | string | **yes**  | EIP-712 types definition as inline JSON or path to a JSON file |
| `--message` | `-m`  | string | **yes**  | Message object as inline JSON or path to a JSON file           |

**Example:**

```
hcli eip712 hash --domain '{"name":"MyApp","version":"1","chainId":295}' --types '{"Mail":[{"name":"from","type":"address"}]}' --message '{"from":"0xAb..."}'

hcli eip712 hash --domain ./domain.json --types ./types.json --message ./message.json
```

**Output:** `{ hash }`

- `hash` — EIP-712 digest (0x-prefixed keccak256)

### `hcli eip712 sign-ecdsa`

Sign an EIP-712 typed data payload using an ECDSA key and output signature components. Only **ECDSA secp256k1** keys are supported.

| Option          | Short | Type   | Required | Default        | Description                                                                                    |
| --------------- | ----- | ------ | -------- | -------------- | ---------------------------------------------------------------------------------------------- |
| `--key`         | `-K`  | string | no       | operator       | Signing key: `accountId:privateKey`, `ecdsa:private:{hex}`, key reference (`kr_xxx`), or alias |
| `--key-manager` | `-k`  | string | no       | config default | Key manager: `local` or `local_encrypted`                                                      |
| `--hash`        | `-H`  | string | no\*     | —              | Pre-computed EIP-712 digest (0x-prefixed hex). Mutually exclusive with typed data options      |
| `--domain`      | `-d`  | string | no\*     | —              | EIP-712 domain as inline JSON or path to a JSON file                                           |
| `--types`       | `-t`  | string | no\*     | —              | EIP-712 types definition as inline JSON or path to a JSON file                                 |
| `--message`     | `-m`  | string | no\*     | —              | Message object as inline JSON or path to a JSON file                                           |

\* Provide either `--hash` OR all three of `--domain`, `--types`, `--message`.

**Example:**

```
hcli eip712 sign-ecdsa --key my-key --domain '{"name":"MyApp","version":"1","chainId":295}' --types '{"Mail":[{"name":"from","type":"address"}]}' --message '{"from":"0xAb..."}'

hcli eip712 sign-ecdsa --key my-key --domain ./domain.json --types ./types.json --message ./message.json

hcli eip712 sign-ecdsa --key my-key --hash 0x<keccak256-hex>
```

**Output:** `{ signerEvm, signature, hash, r, s, v }`

- `signerEvm` — EVM address of the signer (checksum format)
- `signature` — combined 65-byte hex string (`0x` + r + s + v)
- `hash` — EIP-712 digest that was signed (0x-prefixed keccak256)
- `r`, `s` — 32-byte hex components
- `v` — recovery id (27 or 28)

### `hcli eip712 sign-ed25519`

Sign an EIP-712 typed data digest using an Ed25519 key. Only **Ed25519** keys are supported.

| Option          | Short | Type   | Required | Default        | Description                                                                                      |
| --------------- | ----- | ------ | -------- | -------------- | ------------------------------------------------------------------------------------------------ |
| `--key`         | `-K`  | string | no       | operator       | Signing key: `accountId:privateKey`, `ed25519:private:{hex}`, key reference (`kr_xxx`), or alias |
| `--key-manager` | `-k`  | string | no       | config default | Key manager: `local` or `local_encrypted`                                                        |
| `--hash`        | `-H`  | string | no\*     | —              | Pre-computed EIP-712 digest (0x-prefixed hex). Mutually exclusive with typed data options        |
| `--domain`      | `-d`  | string | no\*     | —              | EIP-712 domain as inline JSON or path to a JSON file                                             |
| `--types`       | `-t`  | string | no\*     | —              | EIP-712 types definition as inline JSON or path to a JSON file                                   |
| `--message`     | `-m`  | string | no\*     | —              | Message object as inline JSON or path to a JSON file                                             |

\* Provide either `--hash` OR all three of `--domain`, `--types`, `--message`.

**Example:**

```
hcli eip712 sign-ed25519 --key my-ed25519-key --domain ./domain.json --types ./types.json --message ./message.json

hcli eip712 sign-ed25519 --key my-ed25519-key --hash 0x<keccak256-hex>
```

**Output:** `{ signerPublicKey, hash, signature }`

- `signerPublicKey` — Ed25519 public key of the signer (0x-prefixed hex)
- `hash` — EIP-712 digest that was signed (0x-prefixed keccak256)
- `signature` — 64-byte Ed25519 signature over the digest (0x-prefixed hex)

### `hcli eip712 verify-ecdsa`

Recover the EVM signer address from an EIP-712 ECDSA signature. Optionally assert the recovered address matches an expected signer.

| Option              | Short | Type   | Required | Description                                                                                      |
| ------------------- | ----- | ------ | -------- | ------------------------------------------------------------------------------------------------ |
| `--hash`            | `-H`  | string | no\*     | Pre-computed EIP-712 digest (0x-prefixed hex). Mutually exclusive with typed data options        |
| `--domain`          | `-d`  | string | no\*     | EIP-712 domain as inline JSON or path to a JSON file                                             |
| `--types`           | `-t`  | string | no\*     | EIP-712 types definition as inline JSON or path to a JSON file                                   |
| `--message`         | `-m`  | string | no\*     | Signed message object as inline JSON or path to a JSON file                                      |
| `--signature`       | `-s`  | string | **yes**  | EIP-712 signature to verify — 0x-prefixed 65-byte hex                                            |
| `--expected-signer` | `-e`  | string | no       | Assert recovered address matches: EVM address (`0x...`), Hedera account ID (`0.0.xxx`), or alias |

\* Provide either `--hash` OR all three of `--domain`, `--types`, `--message`.

**Example:**

```
hcli eip712 verify-ecdsa --domain ./domain.json --types ./types.json --message ./message.json --signature 0x...

hcli eip712 verify-ecdsa --hash 0x<keccak256> --signature 0x...

hcli eip712 verify-ecdsa --domain ./domain.json --types ./types.json --message ./message.json --signature 0x... --expected-signer 0xAbCd...1234

hcli eip712 verify-ecdsa --domain ./domain.json --types ./types.json --message ./message.json --signature 0x... --expected-signer 0.0.12345

hcli eip712 verify-ecdsa --domain ./domain.json --types ./types.json --message ./message.json --signature 0x... --expected-signer my-alias
```

**Output:** `{ recoveredSigner, match? }`

- `recoveredSigner` — EVM address recovered from the signature
- `match` — boolean, only present when `--expected-signer` is provided

---

### `hcli eip712 verify-ed25519`

Verify an Ed25519 signature over an EIP-712 digest using a public key from the KMS. Only **Ed25519** keys are supported.

| Option          | Short | Type   | Required | Default        | Description                                                                               |
| --------------- | ----- | ------ | -------- | -------------- | ----------------------------------------------------------------------------------------- |
| `--key`         | `-K`  | string | no       | operator       | Public key to verify against. Key reference, account alias, or account ID                 |
| `--key-manager` | `-k`  | string | no       | config default | Key manager: `local` or `local_encrypted`                                                 |
| `--hash`        | `-H`  | string | no\*     | —              | Pre-computed EIP-712 digest (0x-prefixed hex). Mutually exclusive with typed data options |
| `--domain`      | `-d`  | string | no\*     | —              | EIP-712 domain as inline JSON or path to a JSON file                                      |
| `--types`       | `-t`  | string | no\*     | —              | EIP-712 types definition as inline JSON or path to a JSON file                            |
| `--message`     | `-m`  | string | no\*     | —              | Signed message object as inline JSON or path to a JSON file                               |
| `--signature`   | `-s`  | string | **yes**  | —              | Signature to verify — 0x-prefixed 64-byte hex                                             |

\* Provide either `--hash` OR all three of `--domain`, `--types`, `--message`.

**Example:**

```
hcli eip712 verify-ed25519 --key my-ed25519-key --domain ./domain.json --types ./types.json --message ./message.json --signature 0x...

hcli eip712 verify-ed25519 --key my-ed25519-key --hash 0x<keccak256> --signature 0x...
```

**Output:** `{ signerPublicKey, hash, verified }`

- `signerPublicKey` — Ed25519 public key used for verification (raw hex)
- `hash` — EIP-712 digest that was verified (0x-prefixed keccak256)
- `verified` — boolean, whether the signature is valid for the given key and message

---

## Notes

- `--primary-type` is not a CLI option. ethers.js infers the primary type from the types definition automatically.
- The plugin is stateless — no entries are written to `~/.hiero-cli/state/`.
- ECDSA commands reject Ed25519 keys and vice versa — both throw a `ValidationError`.

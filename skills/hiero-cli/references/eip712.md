# eip712 plugin

Sign and verify EIP-712 structured typed data using ECDSA keys managed by the CLI KMS. Stateless — no local state is written. Useful for permit flows, off-chain authorizations, meta-transactions, and dApp integrations on Hedera.

Only **ECDSA secp256k1** keys are supported. Ed25519 keys throw a `ValidationError`.

Domain, types, and message each accept either an **inline JSON string** or a **path to a JSON file**.

---

### `hcli eip712 sign-ecdsa`

Sign an EIP-712 typed data payload using an ECDSA key and output signature components.

| Option          | Short | Type   | Required | Default        | Description                                                                                    |
| --------------- | ----- | ------ | -------- | -------------- | ---------------------------------------------------------------------------------------------- |
| `--key`         | `-K`  | string | no       | operator       | Signing key: `accountId:privateKey`, `ecdsa:private:{hex}`, key reference (`kr_xxx`), or alias |
| `--key-manager` | `-k`  | string | no       | config default | Key manager: `local` or `local_encrypted`                                                      |
| `--domain`      | `-d`  | string | **yes**  | —              | EIP-712 domain as inline JSON or path to a JSON file                                           |
| `--types`       | `-t`  | string | **yes**  | —              | EIP-712 types definition as inline JSON or path to a JSON file                                 |
| `--message`     | `-m`  | string | **yes**  | —              | Message object as inline JSON or path to a JSON file                                           |

**Example:**

```
hcli eip712 sign-ecdsa --key my-key --domain '{"name":"MyApp","version":"1","chainId":295}' --types '{"Mail":[{"name":"from","type":"address"}]}' --message '{"from":"0xAb..."}'

hcli eip712 sign-ecdsa --key my-key --domain ./domain.json --types ./types.json --message ./message.json
```

**Output:** `{ signerEvm, signature, r, s, v }`

- `signerEvm` — EVM address of the signer (checksum format)
- `signature` — combined 65-byte hex string (`0x` + r + s + v)
- `r`, `s` — 32-byte hex components
- `v` — recovery id (27 or 28)

### `hcli eip712 verify-ecdsa`

Recover the EVM signer address from an EIP-712 signature. Optionally assert the recovered address matches an expected signer.

| Option              | Short | Type   | Required | Description                                                                                      |
| ------------------- | ----- | ------ | -------- | ------------------------------------------------------------------------------------------------ |
| `--domain`          | `-d`  | string | **yes**  | EIP-712 domain as inline JSON or path to a JSON file                                             |
| `--types`           | `-t`  | string | **yes**  | EIP-712 types definition as inline JSON or path to a JSON file                                   |
| `--message`         | `-m`  | string | **yes**  | Signed message object as inline JSON or path to a JSON file                                      |
| `--signature`       | `-s`  | string | **yes**  | EIP-712 signature to verify — 0x-prefixed 65-byte hex                                            |
| `--expected-signer` | `-e`  | string | no       | Assert recovered address matches: EVM address (`0x...`), Hedera account ID (`0.0.xxx`), or alias |

**Example:**

```
hcli eip712 verify-ecdsa --domain ./domain.json --types ./types.json --message ./message.json --signature 0x...

hcli eip712 verify-ecdsa --domain ./domain.json --types ./types.json --message ./message.json --signature 0x... --expected-signer 0xAbCd...1234

hcli eip712 verify-ecdsa --domain ./domain.json --types ./types.json --message ./message.json --signature 0x... --expected-signer 0.0.12345

hcli eip712 verify-ecdsa --domain ./domain.json --types ./types.json --message ./message.json --signature 0x... --expected-signer my-alias
```

**Output:** `{ recoveredSigner, match? }`

- `recoveredSigner` — EVM address recovered from the signature
- `match` — boolean, only present when `--expected-signer` is provided

---

## Notes

- `--primary-type` is not a CLI option. ethers.js infers the primary type from the types definition automatically.
- The plugin is stateless — no entries are written to `~/.hiero-cli/state/`.

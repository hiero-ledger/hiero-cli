# Hedera feature coverage (hiero-cli)

**Purpose:** One-page “visual” gap view of which **Hedera / Hiero** network capabilities are exposed through **hiero-cli** command groups, and what is still missing or only partially covered.

**Status legend**

| Symbol      | Meaning                                                                                          |
| ----------- | ------------------------------------------------------------------------------------------------ |
| **Yes**     | First-class CLI commands cover the capability for typical workflows.                             |
| **Partial** | Supported indirectly, only at creation time, via mirror/query only, or a subset of SDK features. |
| **No**      | No dedicated CLI; would require new commands or wrapping SDK transactions.                       |

---

## Summary by Hedera service area

| Area                             | Capability                                                                      | Status      | Notes / gap                                                                                                                                                                                                                                                                                      |
| -------------------------------- | ------------------------------------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Crypto & fees**                | HBAR transfer (`CryptoTransfer`)                                                | **Yes**     | `hbar transfer`                                                                                                                                                                                                                                                                                  |
| **Crypto & fees**                | Account balance / info queries                                                  | **Partial** | `account balance`, `account view` (mirror-backed where applicable)                                                                                                                                                                                                                               |
| **Crypto & fees**                | HBAR spending allowances (native `approveHbarAllowance` / allowance management) | **No**      | No plugin command; only **HTS** token allowances exist (`token allowance-ft` / `allowance-nft` / `delete-allowance-nft`); tracked in [issue #1729](https://github.com/hiero-ledger/hiero-cli/issues/1729)                                                                                        |
| **Read / query**                 | Transaction receipt or status by user-provided transaction ID                   | **No**      | `api.receipt` is used internally (e.g. hooks after submit); no end-user receipt/get-transaction command in plugin manifests                                                                                                                                                                      |
| **Read / query**                 | Network / node metadata (version, address book) as dedicated commands           | **No**      | Not exposed; users rely on mirror / HashScan out of band                                                                                                                                                                                                                                         |
| **Read / query**                 | Transaction history / blocks listing (“explorer” style mirror browsing)         | **No**      | No commands to list transactions by account, contract, or block range; entity-specific queries only (`account view`, `token view`, `topic find-message`, …)                                                                                                                                      |
| **Accounts**                     | Create / update / delete accounts                                               | **Yes**     | `account create`, `account update`, `account delete`                                                                                                                                                                                                                                             |
| **Accounts**                     | Local wallet / alias bookkeeping                                                | **Yes**     | `account import`, `account list`, `account clear`, credentials helpers                                                                                                                                                                                                                           |
| **Accounts**                     | Staking fields on account (`stakedAccountId`, `stakedNodeId`, `declineReward`)  | **Yes**     | On `account update` (and related options on `contract create`)                                                                                                                                                                                                                                   |
| **HTS — lifecycle**              | Create fungible / NFT tokens                                                    | **Yes**     | `token create-ft`, `token create-nft`, `token create-ft-from-file`, `token create-nft-from-file`                                                                                                                                                                                                 |
| **HTS — lifecycle**              | Mint / burn                                                                     | **Yes**     | `token mint-ft`, `token mint-nft`, `token burn-ft`, `token burn-nft`                                                                                                                                                                                                                             |
| **HTS — lifecycle**              | Delete token                                                                    | **Yes**     | `token delete`                                                                                                                                                                                                                                                                                   |
| **HTS — lifecycle**              | Update token metadata / keys / treasury / expiry after creation (`TokenUpdate`) | **No**      | Keys/treasury/expiry mostly fixed at creation; ongoing token administration not exposed as CLI; tracked in [issue #1650](https://github.com/hiero-ledger/hiero-cli/issues/1650)                                                                                                                  |
| **HTS — lifecycle**              | Update individual NFT metadata per-serial (`TokenUpdateNfts`)                   | **No**      | Introduced in Hedera 0.46; per-NFT metadata patches not exposed as CLI; tracked in [issue #1672](https://github.com/hiero-ledger/hiero-cli/issues/1672)                                                                                                                                          |
| **HTS — lifecycle**              | Update fee schedule (`TokenFeeScheduleUpdate`)                                  | **Partial** | **Custom fee definitions** (`customFees` in JSON) are only on `token create-ft-from-file` / `create-nft-from-file` — not on interactive `token create-ft` / `create-nft` (those set a **fee schedule key** but not the fee list at the CLI). **Post-creation** fee schedule changes → **No** CLI |
| **HTS — custody**                | Associate tokens                                                                | **Yes**     | `token associate`                                                                                                                                                                                                                                                                                |
| **HTS — custody**                | Dissociate tokens (`TokenDissociate`)                                           | **No**      | Not implemented; tracked in [issue #1664](https://github.com/hiero-ledger/hiero-cli/issues/1664)                                                                                                                                                                                                 |
| **HTS — compliance & risk**      | Freeze / unfreeze account on token                                              | **Yes**     | `token freeze`, `token unfreeze`                                                                                                                                                                                                                                                                 |
| **HTS — compliance & risk**      | KYC grant / revoke (`TokenGrantKyc`, `TokenRevokeKyc`)                          | **No**      | KYC keys can be set at creation; ongoing KYC ops → **No** CLI; tracked in [issue #1662](https://github.com/hiero-ledger/hiero-cli/issues/1662) (enable-kyc) and [#1663](https://github.com/hiero-ledger/hiero-cli/issues/1663) (disable-kyc)                                                     |
| **HTS — compliance & risk**      | Wipe (`TokenWipe`)                                                              | **No**      | Wipe keys can be set at creation; wipe operation → **No** CLI; tracked in [issue #1667](https://github.com/hiero-ledger/hiero-cli/issues/1667) (wipe-ft) and [#1668](https://github.com/hiero-ledger/hiero-cli/issues/1668) (wipe-nft)                                                           |
| **HTS — compliance & risk**      | Pause / unpause network transfers (`TokenPause`, `TokenUnpause`)                | **No**      | Pause keys can be set at creation; pause/unpause → **No** CLI; tracked in [issue #1665](https://github.com/hiero-ledger/hiero-cli/issues/1665) (pause) and [#1666](https://github.com/hiero-ledger/hiero-cli/issues/1666) (unpause)                                                              |
| **HTS — transfers & allowances** | FT / NFT transfers                                                              | **Yes**     | `token transfer-ft`, `token transfer-nft`                                                                                                                                                                                                                                                        |
| **HTS — transfers & allowances** | Allowances (approve / approve NFT / delete NFT allowance)                       | **Partial** | `token allowance-ft` (revoke = amount `0` per `allowance-ft` input schema), `token allowance-nft`, `token delete-allowance-nft` (NFT-specific deletions)                                                                                                                                         |
| **HTS — airdrops**               | Airdrop, pending list, claim, cancel, reject                                    | **Yes**     | `token airdrop-ft`, `token airdrop-nft`, `token pending-airdrops`, `token claim-airdrop`, `token cancel-airdrop`, `token reject-airdrop`                                                                                                                                                         |
| **HTS — discovery**              | Token registry in CLI state & on-chain view                                     | **Yes**     | `token list`, `token view`, `token import`                                                                                                                                                                                                                                                       |
| **HCS**                          | Topic create / update / delete                                                  | **Yes**     | `topic create`, `topic update`, `topic delete`                                                                                                                                                                                                                                                   |
| **HCS**                          | Submit message                                                                  | **Yes**     | `topic submit-message`                                                                                                                                                                                                                                                                           |
| **HCS**                          | Browse messages (mirror / filters)                                              | **Yes**     | `topic find-message`                                                                                                                                                                                                                                                                             |
| **HCS**                          | Topic registry in CLI state                                                     | **Yes**     | `topic import`, `topic list`                                                                                                                                                                                                                                                                     |
| **HCS**                          | On-chain topic info (full **get topic** without message scan)                   | **Partial** | No `topic view` command; `topic find-message` + mirror covers message history; use mirror/HashScan for full topic entity details if needed                                                                                                                                                       |
| **Smart contracts**              | Deploy (`ContractCreate` / create flow)                                         | **Yes**     | `contract create`                                                                                                                                                                                                                                                                                |
| **Smart contracts**              | Delete contract                                                                 | **Yes**     | `contract delete`                                                                                                                                                                                                                                                                                |
| **Smart contracts**              | Contract query via mirror (`ContractCall` read-only)                            | **Partial** | Used internally for **ERC-20 / ERC-721** plugins; **no generic “call arbitrary ABI function” command**                                                                                                                                                                                           |
| **Smart contracts**              | Contract execute (`ContractExecute`)                                            | **Partial** | **ERC-20 / ERC-721** command sets (`contract-erc20`, `contract-erc721`) only                                                                                                                                                                                                                     |
| **Smart contracts**              | Contract update (`ContractUpdate`)                                              | **No**      | Admin/auto-renew updates not exposed; tracked in [issue #1643](https://github.com/hiero-ledger/hiero-cli/issues/1643)                                                                                                                                                                            |
| **Smart contracts**              | Ethereum transaction / EVM interoperability transactions                        | **No**      | Not present in codebase survey                                                                                                                                                                                                                                                                   |
| **Scheduling**                   | Schedule create / sign / delete                                                 | **Yes**     | `schedule create`, `schedule sign`, `schedule delete`                                                                                                                                                                                                                                            |
| **Scheduling**                   | Inspect / verify scheduled transaction                                          | **Partial** | `schedule verify` **does** call the mirror (`getScheduled` in `schedule/commands/verify/handler.ts`) and can **update local** schedule state; there is no separate read-only `schedule get` / `schedule info` command                                                                            |
| **Batching**                     | Atomic batch submission patterns                                                | **Yes**     | `batch create`, `batch execute`, `batch list`, `batch delete` (+ hooks on other commands)                                                                                                                                                                                                        |
| **File service**                 | File create / append / update / delete                                          | **No**      | Not exposed in `src/plugins`; tracked in [issue #1645](https://github.com/hiero-ledger/hiero-cli/issues/1645) (epic)                                                                                                                                                                             |
| **Network administration**       | Freeze / prepare upgrade / node admin transactions                              | **No**      | Council / privileged ops not targeted by this CLI                                                                                                                                                                                                                                                |
| **Utilities**                    | PRNG / system contract utilities (`UtilPrng`, etc.)                             | **No**      | No matches in `src` for first-class CLI; not a product focus here                                                                                                                                                                                                                                |
| **Legacy / rare**                | LiveHash add/delete                                                             | **No**      | Not exposed (Hedera has de-emphasized live hash; no CLI support)                                                                                                                                                                                                                                 |
| **CLI platform**                 | Networks, operator, config, plugins                                             | **Yes**     | `network *`, `config *`, `plugin-management *`, `credentials *`                                                                                                                                                                                                                                  |
